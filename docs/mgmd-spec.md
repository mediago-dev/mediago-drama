# mgmd 格式规范(草案 v1)

mgmd(MediaGo Markdown)是 MediaGo Drama 的文档持久化格式。它是 **CommonMark + GFM 的薄超集**:任何 mgmd 文档都是合法的标准 Markdown,可被任意 Markdown 工具打开而不产生乱码;mgmd 感知的应用(编辑器、服务端、MCP、LLM)则能额外识别其中的结构化语义。

## 1. 目标与非目标

**目标**

- 为章节/块提供稳定身份(ID),支撑 @ 引用、评论锚点、素材生成、历史追踪、块级链接。
- 提供**通用的扩展机制**,新功能(自定义块、块属性、行内引用)不再逐个发明语法。
- 三端(前端 TS / Go 服务端 / MCP)解析行为一致,以共享测试语料约束。
- LLM 可直接读写:所有语法在主流训练语料中有充分先例(Pandoc / kramdown / remark-directive)。

**非目标**

- 不做块树 canonical(Notion/飞书模式)。Markdown 文本是唯一事实来源。
- 不承载纯视觉样式(颜色、对齐等)。样式不属于剧本文档的语义。
- 不保证扩展语义在外部渲染器中生效,只保证**优雅降级**(见 §6)。

## 2. 设计原则

1. **标准优先**:扩展语法必须采纳业界已有规范(Pandoc 属性、CommonMark directive 提案),禁止发明私有语法。
2. **身份长在节点上**:元数据以属性形式绑定在节点自身(如标题行内的 `{#id}`),禁止"邻位约定"(依赖相邻行位置关联)。
3. **降级契约**:每个注册扩展必须声明四栏降级行为(§6),缺一不得注册。
4. **未知保留**(向前兼容核心):解析器遇到**未注册**的属性键或 directive 名时,必须原样保留并在序列化时原样输出,不得丢弃或报错。旧版本客户端打开新版本文档不丢数据。
5. **章节 ID 全量化**:二级标题(章节资源边界)在保存时一律自动补 `{#id}`,使"改名/移动不改变身份"成为无条件保证,并淘汰三端各自的派生 ID 算法。其余元数据(非 ID 属性)按需惰性添加。

## 3. 语法分层

### L0 基线:CommonMark + GFM

启用:表格、删除线、任务列表、自动链接。标题使用 ATX 风格(`#` 至 `####`,文档正文标题层级锁定 1–4 级,二级标题为章节资源边界——与现有 DOCUMENT_RULES 一致)。

### L1 文档头:YAML front matter

```markdown
---
mgmd: 1
---
```

- `mgmd` 为保留键,声明规范版本。无 front matter 或无 `mgmd` 键的文档按 v1 + 遗留语法(§8)解析。
- 其余文档级元数据键遵循"未知保留"原则。

### L2 节点属性:`{...}`(Pandoc/kramdown 属性语法)

附着在 ATX 标题行尾:

```markdown
## 1-1 日 内 御灵一中 高三五班教室 {#section_a1b2c3}
```

- 语法:`{#id key=value key2="带空格 value"}`,`#id` 至多一个,置于最前。
- v1 范围:**仅标题节点**。扩展到其他块类型需升级规范(为此保留了语法空间)。
- ID 字面量:`section_[A-Za-z0-9_-]+`,文档内唯一。粘贴/合并产生冲突时,后出现者由编辑器重新生成。
- 键命名空间:无前缀键由本规范保留;实验性键使用 `x-` 前缀。

**注册属性键(v1)**

| 键 | 载体 | 含义 |
|----|------|------|
| `#id` | 标题 | 节点稳定身份 |

### L3 块扩展:directive(CommonMark directive 提案语法)

叶子形式(无内容体):

```markdown
::media{kind="audio" src="/api/resource/xxx" title="第一章配音"}
```

容器形式(含标准 md 内容体,内容体兼作外部渲染降级):

```markdown
:::callout{type="note"}
这里是提示内容,外部渲染器会显示为普通段落。
:::
```

- directive 名:`[a-z][a-z0-9-]*`。未注册名遵循"未知保留"原则。
- 属性括号语法与 L2 一致。

**注册 directive(v1)**

v1 **不注册任何 directive**——机制先行定义,首个使用者(候选:`media` 迁移、`callout`)在 v1.x 增量注册。理由:现有音视频标签链接(§L4)降级表现良好(外部渲染为可点击链接),且 directive 在 Go 生态无原生解析支持,不值得进入 v1 关键路径。

保留名(禁止挪用):`media`、`callout`、`sync`、`comment`。

### L4 标签链接 + 自定义 URI scheme

沿用并收编现有语法——标准链接语法 + 私有 scheme / 标签前缀,本身即合法 Markdown:

```markdown
@[角色-猴子](mention://doc_xxx)
@[1-1 教室](mention://doc_xxx/section_a1b2c3)
@[分镜图](asset://asset_xxx?kind=image&mimeType=image/png)
[章节音频：1-1 配音](</api/resource/ep1-s1.mp3>)
```

章节音视频沿用标签链接形式(`章节音频：` / `章节视频：` 前缀),v1 不迁移;未来若需扩展属性(封面、时长),迁移到 `::media` directive 并将链接形式转为遗留只读。

**注册 scheme(v1)**

| scheme | 含义 |
|--------|------|
| `mention://<docId>[/<sectionId>]` | 文档 / 章节引用 |
| `asset://<assetId>?kind=&mimeType=&url=` | 素材引用 |

新增行内语义优先复用此机制(新 scheme),其次考虑 inline directive(`:name[label]{attrs}`,v1 暂不启用)。

## 4. 解析器实现要求

| 端 | 实现 | 说明 |
|----|------|------|
| 前端 | `@tiptap/markdown`(marked)自定义 tokenizer | 现有 sectionIdAnchor / sectionMediaPreview / documentMention 的同类扩展点 |
| Go 服务端 | goldmark + `parser.WithHeadingAttribute()` | 属性解析用原生能力,directive 用 goldmark 扩展;替换手写正则 |
| MCP | 同 Go(goldmark) | 与服务端共用解析包,消除第三份拷贝 |

**一致性约束**:`docs/mgmd-conformance/` 下维护共享测试语料(输入 md → 期望结构 JSON → 期望序列化输出),三端测试必须跑同一套语料。规范行为以语料为准。

## 5. 剪贴板行为

- `text/html`:节点经 `renderHTML` 携带 `data-*` 属性(含 section id),粘回 mgmd 应用完整还原。
- `text/plain`:纯文本,所有扩展语义蒸发——粘贴到外部文本编辑器自然丢失元数据(预期行为)。
- 粘贴进入 mgmd 应用时,对重复 ID 执行重新生成(保留首个)。

## 6. 降级矩阵(注册扩展必填)

| 扩展 | mgmd 应用内 | 标准 md 渲染器 | 纯文本 | 导出标准 md 时的转换 |
|------|------------|---------------|--------|---------------------|
| `{#id}` 标题属性 | 章节身份,不可见 | 标题尾部小段文字(轻噪音) | 同左 | 剥除 |
| `[章节音频：…](<src>)` 标签链接 | 播放器卡片 | 可点击链接 | 字面文本 | 保持原样 |
| `@[...](mention://…)` | 引用胶囊 | `@` + 死链接(可见标题) | `@[标题](uri)` | 转为纯文本标题 |
| `::media` / `:::callout`(预留) | — | 一行字面文本 / 属性行噪音+内容体 | 同左 | 注册时补充 |

「导出标准 md」是一个显式的降级转换器,用户外发文档时一键执行。

## 7. 版本与演进规则

- **新增**属性键 / directive 名 / scheme:不升版本。旧解析器按"未知保留"原则安全通过。
- **变更或移除**已注册语义、或引入旧解析器无法保留的语法:升 `mgmd` 主版本,front matter 声明。
- 注册即承诺:进入注册表的扩展需同时提交降级矩阵行 + 三端一致性语料。

## 8. 遗留语法(只读兼容)

解析器长期保留识别,序列化一律输出新语法(文档被编辑一次即自然迁移):

| 遗留语法 | 对应新语法 |
|---------|-----------|
| `<!-- section-id: section_xxx -->`(标题前注释行) | 标题 `{#section_xxx}` 属性 |
| `section-<hash>` 旧格式 ID / 各端派生 ID | 全量 `{#id}` 后退役,仅保留只读识别 |

**发布顺序约束**:三端"双读"版本必须先发布并覆盖存量客户端,之后才允许切换写入端(桌面应用需隔至少一个版本周期)。

## 9. 示例文档

```markdown
---
mgmd: 1
---

# 第一章 御灵之门

## 1-1 日 内 御灵一中 高三五班教室 {#section_a1b2c3}

人物:@[林墨](mention://doc_chars/section_linmo)、叶初夏(OS)

下课铃响过,教室里仍是嗡嗡一片。

[章节音频：1-1 配音](</api/resource/ep1-s1.mp3>)

## 1-2 日 内 御灵一中 高三五班教室 {#section_d4e5f6}

黑靴踏进门,高挑的黑色制服身影停在讲台前。
```
