# Skill 浏览双栏与导入内容隔离设计

## 目标

- Skill 浏览页取消下拉选择，改为左侧完整 Skill 列表、右侧当前 Skill 详情。
- 导入技能包的 Skill 名称出现在左侧列表中，但条目禁用，也不能通过 Skill 详情接口读取。
- 已启用导入包仍可被 Agent 运行时加载；启停和卸载能力保持不变。

## 数据边界

运行时继续使用 `skill.Registry.List` 和 `skill.Registry.Get`，其中包含已启用导入包内容。HTTP 列表接口使用 `ListBrowsable`：默认包和本地包返回正常元数据，导入包只返回名称、所属包和来源等列表必需字段。详情接口使用 `GetBrowsable`，拒绝读取导入 Skill。这样不会把名称可见性、内容查看权限和运行时装载能力混在一起。

前端只消费浏览接口返回的数据。Skill 页面复用提示词库的双栏网格、圆角边框和左侧选中行样式；列表按右侧所属技能包 Tag 分组，默认包、本地包、导入包依次排列，导入 Skill 统一置底。导入 Skill 行显示但禁用，其他 Skill 选择后在右侧加载描述与 Markdown 正文。创建 Skill 时仅提供默认包和本地包作为目标。

## 验证

- 前端测试覆盖无下拉框、双栏结构、多 Skill 切换，以及导入 Skill 可见但不可选择。
- 后端测试覆盖运行时列表仍含导入 Skill、浏览列表仅返回其安全索引字段，以及浏览详情拒绝导入 Skill。
- 运行前端 lint、格式、构建与后端 race 测试。
