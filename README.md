## 项目简介

MediaGo Drama 是一个围绕短剧内容生产流程设计的 AI 创作工作台。它将剧本创作、素材组织、图像与视频生成、MCP 工具调用以及 Agent 协作能力整合在同一个本地优先的工作环境中，帮助创作者和业务团队更高效地完成从创意构思到内容资产沉淀的全过程。

与单点式的生成工具不同，MediaGo Drama 更关注短剧项目的连续创作链路。项目中的故事设定、人物关系、分集剧本、场景素材、生成结果和工具调用上下文都可以被持续组织、复用和迭代，从而降低重复整理成本，让 AI 能力更自然地进入实际生产流程。

## 项目定位

MediaGo Drama 面向需要高频创作、快速迭代和持续沉淀内容资产的短剧业务场景。它既可以作为个人创作者的本地创作台，也可以作为团队内部探索 AI 短剧生产流程的业务基础设施。

项目重点解决以下问题：

- 将短剧创作中的剧本、角色、场景、素材和生成结果统一管理。
- 让 AI 生成能力围绕具体创作流程展开，而不是停留在零散提示词调用。
- 通过 MCP 工具扩展外部能力，使创作工作台可以连接更多业务工具和模型服务。
- 通过 Agent 协作机制，将大纲扩写、剧本润色、素材整理、生成任务规划等工作拆分给不同智能体处理。
- 以本地优先的方式保留项目上下文，提升创作数据和素材资产的可控性。

## 核心能力

### 剧本创作

- 支持围绕短剧项目组织创意、故事大纲、人物设定、场景设定和分集内容。
- 帮助创作者在同一项目上下文中持续迭代剧情结构、人物关系和对白表达。
- 适用于从早期脑暴到剧本文本细化的连续创作过程。

### 素材管理

- 管理角色、场景、参考图、生成结果等短剧制作过程中常用的创作资产。
- 将素材与具体项目上下文关联，减少跨工具查找、复制和整理的成本。
- 支持在后续创作和生成任务中复用已有素材与设定。

### 图像与视频生成

- 面向短剧生产流程组织图像、视频及相关生成任务。
- 将生成结果沉淀为项目资产，便于后续筛选、复用和迭代。
- 为角色视觉、场景概念、分镜素材和视频片段等内容生产环节提供统一入口。

### MCP 工具集成

- 支持通过 MCP 接入外部工具、模型能力和自动化流程。
- 让创作工作台可以根据业务需要扩展新的工具链，而不局限于固定功能。
- 适合承载检索、生成、转换、分析、文件处理等可插拔能力。

### Agent 协作

- 支持通过 Agent 参与短剧创作、素材整理和任务规划。
- 将复杂创作流程拆解为更明确的智能体任务，提升多步骤工作的可控性。
- 为后续扩展自动化创作、审校、生成编排和内容管理流程提供基础。

## 使用场景

- 短剧项目立项阶段的创意整理、题材探索和故事雏形构建。
- 分集剧本、人物设定、场景设定和对白内容的持续迭代。
- 角色图、场景图、参考图、视频片段等素材的集中管理。
- 多模型、多工具、多 Agent 共同参与的 AI 创作流程探索。
- 内部短剧生产流程、提示词经验、生成结果和项目资产的长期沉淀。

## 设计原则

- **本地优先**：优先保障创作数据、素材资产和项目上下文在本地可控。
- **流程聚合**：围绕短剧创作链路组织功能，减少零散工具之间的切换成本。
- **资产沉淀**：让剧本、角色、场景、参考素材和生成结果可以被持续复用。
- **能力可扩展**：通过 MCP 和 Agent 机制接入新的工具能力、模型能力和自动化流程。
- **面向业务迭代**：功能设计服务于真实短剧生产过程，并随业务反馈持续调整。

## 项目状态

MediaGo Drama 当前处于业务迭代阶段。项目能力会随着短剧生产流程、AI 生成能力、MCP 工具生态和 Agent 协作方式的演进持续调整。
