<div align="center">

**让 Agent 7×24 小时持续推进从小说到漫剧成片的生产流程。**

MediaGo Drama 正在构建一个可长期运行的自主创作 Agent：从原文解析、剧本改写到角色、场景、道具、分镜和视频生成，Agent 在同一个项目上下文中连续工作，并把过程产物保存为本地文档。

## 为什么使用 MediaGo Drama

用普通聊天工具制作漫剧时，原文分析、角色设定、场景规则和分镜提示词往往散落在不同对话里。随着内容增多，创作者需要反复补充背景，已经确认的设定也容易在后续生成中丢失。

MediaGo Drama 将整个创作过程组织成一个本地项目：

- **过程文档完整保留**：原文、剧本、角色、场景、道具和分镜以可读、可编辑的文档持续保存。
- **Agent 理解项目上下文**：Agent 可以读取已有文档、引用具体设定、调用 Skills，并把结果写回项目。
- **创作资产持续复用**：角色形象、场景描述、关键道具和参考素材可以跨章节、跨分镜继续使用。
- **文档直接进入生成流程**：已经确认的角色、场景和分镜可以继续用于图片与视频生成，不必重新拼装 Prompt。
- **方法能够沉淀下来**：团队可以把有效的写作规则和提示词整理成 Skills 与模板，而不是留在一次性的聊天记录里。
- **为无人值守而保存状态**：本地文档既是创作资产，也是 Agent 的长期记忆、任务检查点与故障恢复依据。

创作者可以只定义题材、目标、约束、预算和验收标准，让 Agent 持续拆解并执行任务；需要时仍然可以打开文档检查或介入，但不必守在每一个生成环节前。

## 创作流程

| 阶段 | 项目产物 | 在自动工作流中的作用 |
| --- | --- | --- |
| 1. 导入原文 | 小说、故事梗概、已有剧本 | 建立 Agent 后续任务共享的事实来源 |
| 2. 解析剧情 | 事件顺序、人物关系、场景切换、关键道具 | 自动形成结构化故事上下文 |
| 3. 改写剧本 | 分场剧本、动作描写、对白 | 按题材规则推进节奏、冲突和人物动机 |
| 4. 建立角色 | 外观、服饰、身份、识别特征 | 为跨镜头生成提供稳定的视觉约束 |
| 5. 建立场景与道具 | 环境设定、关键物件、生成提示词 | 形成后续任务可反复调用的世界观资产 |
| 6. 拆解分镜 | 镜头组、动作、运镜、光影、台词、时长 | 自动转换成可执行的图片与视频任务 |
| 7. 生成素材 | 角色图、场景图、道具图、镜头参考图 | 批量生成、归档结果并回写任务状态 |
| 8. 生成与预览视频 | 分镜视频、分集预览、生成历史 | 汇总结果，并把异常与质量信息送入下一轮 |

每个阶段都可以配置为自动执行或人工确认节点。后面的生成结果不会覆盖前面的创作过程，而是与原文、设定、分镜、任务状态和失败记录一起留在项目中，让 Agent 能够继续运行或从检查点恢复。

## 核心能力

### 文档驱动的本地创作工程

- 项目文档优先以 Markdown 等本地文件保存，不被封闭在某一次 Agent 会话中。
- 文档同时承担长期记忆、任务状态、质量依据和恢复检查点的作用。
- 创作者可以阅读、搜索、手动编辑并长期归档，自动任务也可以直接继续使用。
- 新会话可以基于已有文档续作，无需重新复述全部角色关系和世界观。
- Agent 重启、任务中断或更换生成模型后，已经整理好的创作资产仍然可以继续使用。

> 本地优先不等于完全离线。项目文档与创作资产保存在本机；图片、视频和文本生成是否访问外部服务，取决于用户选择的模型与供应商。

### 面向项目的 Agent 工作台

- Agent 可以读取当前项目、引用指定文档并调用 MCP 工具。
- 支持按任务装载剧本、角色、场景、道具和分镜 Skills。
- 长任务的探索、工具调用和文档修改过程可以在工作台中查看。
- 创作者定义目标、约束、预算和验收标准，Agent 负责拆解依赖并持续执行生产任务。
- 产品方向是逐步减少人工确认，让 Agent 能够在规则和资源边界内自主续跑。

### 结构化的漫剧资产

- **剧本**：场景标题、动作描写、对白和戏剧推进。
- **角色**：外观、身份气质、服饰道具和跨镜头识别特征。
- **场景**：环境、构图、光影、材质与可直接用于生图的 Prompt。
- **道具**：剧情功能、外观材质、使用关系和连续性标记。
- **分镜**：主体、动作、场景、运镜、光影、台词和时长。

这些内容既是给模型使用的提示信息，也是创作者能够检查和修改的生产资料。

### 图片、视频与素材管理

- 围绕项目中的角色、场景、道具和分镜生成图片与视频。
- 支持参考素材、生成规格、模型路由和批量生成设置。
- 生成结果进入项目素材与历史记录，便于筛选、回看和再次引用。
- 分镜视频可以进入分集预览，集中检查已生成与待生成镜头。

### 可编辑的 Skills 与提示词

剧本、角色、场景、道具和分镜等创作规则以 Skill 文件管理。内置规则可以直接使用，也可以根据团队题材、画风和生产规范进行调整。

| Skill | 作用 |
| --- | --- |
| `screenplay-writer` | 把小说叙述改写成分场剧本、动作与对白 |
| `character-writer` | 提取可成像的角色设定与跨镜头识别特征 |
| `scene-writer` | 生成纯净、可复用的场景设定与生图 Prompt |
| `prop-writer` | 梳理关键道具的剧情功能、视觉细节和连续性 |
| `storyboard-writer` | 把小说或剧本拆成面向视频模型的镜头组 |
| `novel-writer` | 撰写、续写、改写和润色小说正文 |
| `image-generation` | 编排图片模型、参数、参考图与异步任务提交；提交后由后台完成、落库并通知 |
| `video-generation` | 编排视频模型、参数、首帧参考与异步任务提交；提交后由后台完成、落库并通知 |

项目还内置角色概念图、多视图、场景氛围图、场景四视图和电影感镜头等提示词模板，并提供真人写实、2D 动漫、3DCG 动漫和 Q 版等风格预设。

### 模型路由与扩展

- 图像、视频和文本生成由统一的模型目录与参数系统管理。
- 模型家族与执行渠道分离，便于根据效果、成本和可用性选择生成路线。
- 通过 MCP 扩展文档工具、外部服务和自动化能力。
- 支持 Codex、Opencode 等 Agent 运行时，方便尝试不同协作方式。

## 快速开始

当前版本主要面向本地开发与工作流体验。

### 环境要求

| 工具 | 建议版本 | 说明 |
| --- | --- | --- |
| Node.js | 24 | 前端与脚本运行环境，与发布构建保持一致 |
| pnpm | 11.9+ | 包管理工具 |
| Go | 1.25+ | 服务端与 MCP 模块 |
| go-task | 3.x | 项目任务编排 |

### 安装依赖

```bash
pnpm install
```

### 启动开发环境

开发环境需要同时启动服务端和 Electron 桌面端。分别打开两个终端：

```bash
# 终端 1：启动本地服务端
pnpm dev:server
```

```bash
# 终端 2：启动 Electron 桌面端
pnpm dev:desktop
```

修改服务端或 Agent MCP 代码后，需要运行 `pnpm build:server` 更新相关二进制。

### 常用命令

```bash
pnpm build:server     # 构建服务端与 Agent MCP 二进制
pnpm workspace:dev   # 仅启动前端工作台
pnpm dev:server      # 启动本地服务端
pnpm dev:desktop     # 启动 Electron 桌面端
pnpm build           # 构建完整项目
pnpm check:go        # 检查 Go packages
```

## 项目状态

MediaGo Drama 当前处于持续迭代阶段，已经具备本地项目管理、文档编辑、Agent 协作、内置 Skills、提示词模板、素材管理以及图片、视频、文本生成工作区，为长期自主运行建立了项目状态与工具基础。

产品的明确目标是成为可以 7×24 小时无人值守出片的漫剧生产 Agent。当前版本的部分节点仍可能需要人工确认；后续会重点完善自动编排、质量验收、失败恢复、成本控制和批量生产能力，逐步把整条工作流交给 Agent 持续运行。
