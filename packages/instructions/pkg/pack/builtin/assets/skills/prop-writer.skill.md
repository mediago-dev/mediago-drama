# 类型专属写作指导：道具设定（prop）

## 核心目标

从剧情、剧本、角色和场景上下文中提炼关键道具，写成可被拍摄、美术、分镜和图像/视频生成复用的稳定设定。每个道具都要说明它是什么、长什么样、怎么被角色使用、在剧情中承担什么功能，以及跨镜头必须保持一致的识别细节。

## 先定风格

- 动笔前优先调用 MCP `get_project_config`，读取 `config.overview.style` 确定项目视觉风格；不要在当前工作目录或父目录中搜索 `project.media.json`。
- 道具描述必须贴合项目视觉风格：真人写实要强调材质、磨损、尺度和可拍细节；动漫/3DCG/Q版要强调造型轮廓、配色块面和可识别符号。
- 如果项目视觉风格未设定，先询问用户确认风格；不要默认某一种风格后继续产出。

## 写作重点

- **剧情功能**：说明道具推动情节、揭示身份、制造误会、提供线索或承担动作的方式。
- **可成像细节**：给出轮廓、材质、颜色、尺寸、状态、磨损、标记、光泽等图像模型能解析的视觉信息。
- **使用关系**：写清楚谁持有、谁使用、出现在哪些场景、与角色服饰或场景摆放的关系。
- **连续性标记**：列出跨镜头不可改变的独特细节，避免生成时颜色、材质、大小或标识漂移。
- **安全边界**：涉及武器、药品、危险装置等高风险道具时，只描述叙事和视觉设定，不提供现实制作、伤害或规避监管细节。
