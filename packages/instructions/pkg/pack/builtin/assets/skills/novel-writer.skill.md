# 类型专属写作指导：小说正文与资料（reference）

## 核心目标

写出可读、可续、可拆解的小说文本。正文要有明确叙事视角、场景目标、人物欲望、冲突推进和情绪落点；同时为后续剧本、角色、场景、道具和分镜拆解保留清晰线索。

## 先读上下文

- 动笔前先读取当前项目已有的资料、剧本、角色、场景、道具和分镜，确认世界观、主角目标、人物关系、时间线和已发生事件。
- 续写时必须接住上一段最后的动作、情绪和悬念，不要跳过当前场景的因果链。
- 改写或润色时保留原文核心事件、人物关系和关键信息，除非用户明确要求重构剧情。
- 如果用户只给出题材或一句灵感，先补一个短大纲再写正文；不要直接写成散乱片段。

## 写作原则

- **视角稳定**：每个场景优先使用一个清晰视角，避免频繁跳头。需要切换视角时用章节、场景分隔或明确转场。
- **场景驱动**：每个场景都要有目标、阻碍、行动、结果。不要只做背景说明或设定堆叠。
- **人物有欲望**：角色说话和行动要服务其当前目标、恐惧、误解或隐瞒。对白不要替作者解释设定。
- **展示而非告知**：用动作、物件、环境反应和选择呈现信息，少用抽象评价。
- **节奏有钩子**：章节开头给出问题或张力，段落中持续推进，结尾留下新信息、反转、危险或情绪余波。
- **可拆解**：关键人物、地点、道具、冲突和伏笔要具体，便于后续提取成角色设定、场景设定、道具设定和分镜。

## 长文工作流

1. 先写或确认本章目的：主角想要什么、遇到什么阻碍、结尾状态如何变化。
2. 列出 3 到 7 个剧情节拍，每个节拍只承载一个动作或信息转折。
3. 按节拍写正文，保持时间顺序和因果关系。
4. 写完后检查连续性：人物称呼、地点、时间、伤势、道具位置、已知信息是否前后一致。
5. 需要多章输出时，先给章节目录和每章一句话钩子，再分章写，不要一次性糊成超长文本。

## 文风控制

- 用户指定文风、题材、平台或受众时，优先服从用户要求。
- 未指定时，默认使用适合短剧改编的中文网文叙事：句子清楚、动作明确、信息密度高，悬念和反转可见。
- 玄幻、都市、悬疑、言情、科幻等类型要遵守类型期待，但不要机械套模板。
- 高风险、露骨或平台敏感内容要弱化为可过审表达，同时保留剧情关系和人物情绪。

## 输出要求

- 直接输出可保存进资料文档的 Markdown 正文，不解释创作过程。
- 章节用一级标题，场景或小节用二级标题；普通正文使用自然段。
- 需要保存设定时可在文末加入“创作备注”，但不要把备注混进正文。
- 不要写剧本格式、分镜字段或生图 Prompt，除非用户明确要求。

## 模板格式

新建或填写资料中的小说文档时，优先使用以下骨架。可以根据用户要求删减，但章节正文必须保持可读的小说形态。

```markdown
# 第 1 章 章节标题

## 场景一：地点或事件

正文段落。用具体动作、环境反应和人物选择推进剧情。

正文段落。保持视角稳定，接住上一段的情绪和动作。

## 场景二：地点或事件

正文段落。引入新的阻碍、误会、线索或反转。

正文段落。以状态变化、悬念、危险或情绪余波收束。

## 创作备注

- 本章目的：
- 关键人物：
- 关键场景：
- 关键道具：
- 伏笔/待回收：
```
