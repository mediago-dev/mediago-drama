# 类型专属写作指导：场景设定（scene）

## 核心目标

根据用户提供的剧情、剧本或场景提示，生成纯净且具有唯一性的场景设定。场景文档只呈现关键场景本身，不输出场景提取清单，不使用固定字段表格。

## 风格处理原则

- 不因缺少视觉风格而暂停、追问或要求用户先选择风格；先完成场景文档。
- 若用户本轮明确指定风格，场景描述必须贴合该风格：真人写实强调真实空间、材质、光线和摄影可呈现细节；动漫、3DCG、Q 版强调轮廓、色块、空间符号和风格化质感。
- 若用户未指定风格，使用风格中性的基础场景描述：空间结构、时间、光线、材质、核心物件、前中后景和氛围；不要写死镜头、媒介、渲染方式或画风。
- 图像生成环节可能会自动追加风格预设；本场景文档只需保证空间设定清晰、无人且可复用。

## 写作重点

- **只写关键场景**：根据剧情自动提取最需要建模、绘制或复用的空间；每个场景使用二级标题。
- **命名有辨识度**：场景标题要具体，避免只写“房间”“街道”“办公室”等单一名词。
- **纯净无人**：画面中严禁出现人物、人影、角色姓名、字幕、水印或 UI 元素。
- **四类信息完整**：场景内容应自然覆盖环境类型、具体时间、空间氛围和视觉主要特征，但不要套固定字段。
- **Prompt 可复制**：每个场景必须包含可直接用于生图的 Prompt；Prompt 要强调 `不能出现其他人, 无人, 纯场景`，并包含 `no humans, empty, landscape only` 等约束。

## 避免事项

- 不输出 `场景提取清单`。
- 不使用固定字段表、空字段或占位符。
- 不在 Prompt 中提及角色姓名、人物动作或剧情解释。
- 不把场景写成抽象气氛词堆叠；必须落到空间结构、材质、光线、核心物件和前中后景元素。
