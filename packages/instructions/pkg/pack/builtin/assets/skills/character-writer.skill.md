# 类型专属写作指导：AI 图像外观描述词（character）

## 核心目标

通读并梳理全文小说剧情与人物线，提炼每个角色的外观设定，按项目所选视觉风格转写为可直接供 GPTImage2 等 AI 图像工具参考的外观描述词，确保同一角色跨镜头视觉一致。

## 先定风格

- 动笔前优先调用 MCP `get_project_config`，读取 `config.overview.style` 确定项目视觉风格；不要在当前工作目录或父目录中搜索 `project.media.json`。当前四种预设为 `真人写实`、`2D动漫`、`3DCG动漫`、`Q版风格`。
- 外观描述词必须贴合该风格：`真人写实` 使用真实选角和摄影可呈现的具体特征；`2D动漫` 使用 2D 日系或国风人设特征；`3DCG动漫` 使用 3D 渲染质感和造型语言；`Q版风格` 使用大头小身、高辨识度、简化造型等 Q 版特征。
- 只需贴合所选风格的措辞、比例和质感，不必照抄风格预设整段；图像生成时会自动追加风格预设。
- 若 `config.overview.style` 未设定，必须先在 Agent 面板直接询问用户从 `真人写实`、`2D动漫`、`3DCG动漫`、`Q版风格` 中选择一个；不要默认真人写实，也不要在用户确认风格前继续产出角色外观描述。

## 写作重点

- **通读锁线索**：基于全文剧情、对话、旁白中的外观信息，结合身份、年龄、经历做自洽推断；原文缺失处给出合理补全。
- **可成像**：使用图像模型可解析的具体特征，例如脸型、五官、肤色、发型发色、身材、服饰、随身物，少用抽象或主观评价。
- **结构自定**：每个人物内部结构由内容需要决定，可以使用自然段、短列表或加粗小标题；不要套固定字段，不要输出空字段。
- **视觉一致**：同一人物的年龄、脸型、发型、体态、服装和识别细节不能自相矛盾；需要锁定 identity 的特征要写清楚，便于多图保持同一人。

## 避免事项

- 不把人物设定写成剧情梗概或单纯性格标签，例如“勇敢”“腹黑”“善良”。
- 不堆空泛形容词，例如“绝美”“高级”“有气场”；要落到可被图像模型识别的视觉细节。
- 不强制使用 `形象定位`、`面部特征` 等固定字段；如果使用小标题，也必须根据人物内容自然组织。
- 不写死某一种风格，尤其不要默认真人写实；一切以 `config.overview.style` 为准。
